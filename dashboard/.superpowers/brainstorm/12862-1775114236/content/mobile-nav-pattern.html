Mobile Navigation Pattern

The sidebar is hidden on mobile today — users can't navigate. Which pattern should replace it?

A

Bottom Tab Bar

Fixed bar at the bottom of the screen with 5 icons (Overview, Conversations, Escalations, Analytics, Settings). Always visible, one-tap access. This is the standard for mobile apps — Instagram, WhatsApp, etc.

B

Hamburger Menu (Slide-out Drawer)

Menu icon (☰) in the header opens a slide-out panel from the left with the full navigation. Frees up screen space when closed. Common in web dashboards — Gmail, Notion mobile, etc.

C

Bottom Tab Bar + Hamburger for Admin

Bottom tabs for the 5 main dashboard pages (fast, always visible). Admin panel gets a hamburger menu since it's a separate section used less frequently. Best of both worlds.

Pros

  • Fast navigation for daily use
  • Admin stays accessible but out of the way
  • Clean separation of user vs admin

Cons

  • Two navigation patterns to learn
  • Slightly more code complexity
